## CPU Sensor Issue Diagnosis

For the provided live snapshot pattern:

- CPU load is valid.
- CPU temperature is unavailable because the only live candidate, Core (Tctl/Tdie), reports 0 C and is rejected as invalid.
- CPU power is treated as stale-zero/unavailable when it reports 0 W while CPU load is nonzero.
- CPU clock is treated as stale-zero/unavailable when it reports 0 MHz while CPU load is nonzero.
- CPU voltage is valid when a reasonable VID voltage is exposed.
- Windows performance counters are used as a lower-ranked fallback for CPU load/frequency when available.

This makes the app show useful CPU telemetry without pretending that 0 W or 0 MHz are real running values.

## Provider Pipeline Fix

The app no longer treats LibreHardwareMonitor as the only CPU source. A Sensors refresh now attempts:

- LibreHardwareMonitor
- HWiNFO shared memory probe
- Windows performance counters
- WMI/CIM thermal data
- ACPI thermal zones
- nvidia-smi
- PresentMon readout

Each provider reports enabled, attempted, available, status, reason, probe time, sensor count, valid sensor count, and rejected sensor count. HWiNFO is reported precisely as not running, shared memory unavailable/not configured, or detected but not safely parsed. Windows counters can provide CPU load and CPU frequency fallback values. WMI/CIM and ACPI thermal data remain low-confidence diagnostic thermal sources and are not labeled CPU die/package temperature unless the source identifies itself as CPU-like.

## GPU Classification

Phase 8 now classifies GpuNvidia, GpuAmd, hardware types starting with Gpu, NVIDIA/RTX/GeForce text, AMD Radeon text, and GPU text into GPU categories/subcategories. GPU Memory Junction 255 C is marked invalid_value.

## Known Issues

- CPU temperature may remain unavailable on the AMD Ryzen AI platform if LibreHardwareMonitor only exposes Core (Tctl/Tdie) as 0 C.
- CPU power and clock may remain stale-zero through LHM on this platform.
- HWiNFO shared memory is probed, but a full shared-memory row parser is still guarded; if HWiNFO is not running or shared memory is unavailable, the app reports that exact reason.
- WMI/CIM and ACPI thermal providers may expose no useful thermal rows on this machine.

## Recommended Phase 9

Add a full safe HWiNFO shared-memory row parser if HWiNFO shared memory is enabled, plus user-controlled sensor calibration tools. Keep all tuning writes blocked. Focus on HWiNFO readout, manual sensor pinning improvements, sensor naming overrides, and exportable telemetry diagnostics. Do not enable MSI, NVIDIA, fan, EC, service, startup, scheduled task, registry, or automatic game-apply writes.
